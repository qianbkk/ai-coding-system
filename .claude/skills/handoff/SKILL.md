# Skill：handoff — Session 交接与续接

**触发条件：**
- Context 使用超过 60%（感觉 session 变长了）
- 任务需要跨天或跨 session 完成
- 用户说"先停在这，明天继续"
- 收到 `/compact` 提示
- 用户粘贴了 HANDOFF.json 或 handoff.md 要求续接

**适用路径：** 简单路径 + 复杂路径均适用

---

## 两种模式

### 模式 A：生成交接文档（Session 即将结束时）

### 模式 B：从交接文档续接（新 Session 开始时）

---

## 模式 A：生成交接文档

在 session 结束前，输出以下内容并保存到文件：

**保存位置：** `.claude/handoff/HANDOFF-YYYY-MM-DD-HH.md`

```markdown
# Session 交接文档
_生成时间：YYYY-MM-DD HH:mm_
_任务：[任务名称]_

## 进度概览
- 阶段：[clarify / spec / plan / execute / review / debug]
- 整体进度：[X%]，[X/Y 个任务]

## 已完成
- [T01] [任务名] ✅ — git commit: [hash 或描述]
- [T02] [任务名] ✅ — git commit: [hash 或描述]

## 进行中（中断点）
- [T03] [任务名] 🔄
  - 已完成的部分：[描述]
  - 中断位置：[具体行/函数/文件]
  - 下一步：[具体要做什么]

## 待完成
- [T04] [任务名]
- [T05] [任务名]

## 关键决策（已做的设计决定）
- [决策 1]：选择了 [X] 而不是 [Y]，原因是 [Z]
- [决策 2]：...

## 已知问题和 Gotcha
- [问题 1]：[描述 + 当前状态]
- [注意点]：[下次继续时需要注意的事]

## 环境状态
- 当前分支：[branch name]
- 未提交的改动：[描述或 git status 摘要]
- 测试状态：[通过 / X 个失败]

## 续接指令（复制给新 Session 使用）
> 我在继续一个中断的任务。请读取 .claude/handoff/HANDOFF-YYYY-MM-DD-HH.md
> 从 [T03] 的中断点继续，下一步是 [具体操作]。
```

---

## 模式 B：从交接文档续接

收到交接文档后，**先输出一个简短的续接确认，再开始工作：**

```
📋 续接确认

读取了交接文档：[文件名/描述]

当前状态：
- 已完成：[列表]
- 进行中：[任务名]，上次中断在 [位置]
- 待完成：[X 个任务]

关键背景：[2-3 句最重要的上下文]

接下来我会：[具体的下一步操作]

确认后继续？还是有需要调整的地方？
```

等用户确认后再开始执行。

---

## Context 预警

当你感觉 session 变长，质量可能下降时，主动告知：

```
⚠️ Context 提示

我们的 session 已经比较长了，继续可能影响后续回复质量。

建议：
1. 我先生成交接文档（2 分钟）
2. 你开一个新 session，粘贴续接指令
3. 新 session 的回复质量会更好

或者，如果任务快完成了，我们可以继续推进到完成。

你希望怎么做？
```

---

## 轻量版交接（简单任务）

如果任务较小，可以用文字替代文件：

```
进度备注：
已完成：[描述]
正在：[描述]
下一步：[具体操作]
注意：[重要的 Gotcha]
```

粘贴给新 session 即可续接。
